# 🛠️ رحلة بناء وثّق — سجل التطوير

توثيق تسلسلي لرحلة بناء منصة **وثّق** من تصميم ثابت إلى منتج حيّ كامل
(واجهة + قاعدة بيانات + ذكاء اصطناعي حقيقي).

**التاريخ:** ٣٠ يونيو ٢٠٢٦ · **المطوّر:** alialq146 · **الرابط:** https://wathiq-ai.vercel.app

---

## نظرة عامة

بدأنا من **حزمة تصميم** (نماذج HTML/CSS مُصدّرة من أداة تصميم) وانتهينا بتطبيق
**Next.js + TypeScript** منشور على Vercel، مربوط بقاعدة بيانات **Neon Postgres**،
ويستخدم **Claude (Opus 4.8)** لتحليل وثائق المتطلبات فعليًا.

### المرحلة ٩ — تحليل وثّق بالذكاء الاصطناعي

- **مخرجات منظّمة (Structured Outputs)** لاستخراج المتطلبات بدقة.
- لصق وثيقة → استخراج المتطلبات + ملخّص شفّاف + خطوات الاستنتاج + توصيات + درجة ثقة.
- زر **«حفظ في القاعدة»** يضيف المستخرجة إلى Neon.
- المفتاح يُقرأ من متغيّر البيئة `ANTHROPIC_API_KEY` (لا يدخل الكود أبدًا).

### المرحلة ١٥ — تسجيل الدخول

- **آمن بالتصميم:** المصادقة معطّلة ما لم تُضبط متغيّرات البيئة، فلا تُقفل الموقع بالخطأ.
- تحقّق شامل: تحويل غير المُصادَق إلى `/login`، رفض/قبول بيانات الدخول، ١١ اختبار وحدة، ولقطة شاشة للشاشة.

### المرحلة ١٦ — نظام الحسابات الكامل (v1.0.0)
- جدول **User** + صفحة **إنشاء حساب** `/signup` — أي شخص يسجّل ويعمل في مساحته.
- كلمات المرور مشفّرة بـ **scrypt**، والجلسات بـ Cookie موقّع يحمل هوية المستخدم.
- **عزل البيانات:** `ownerId` على كل الجداول؛ كل حساب يرى بياناته + التجريبية المشتركة، وحُماية الملكية مفروضة في كل Server Action.
- سجل التدقيق يوثّق العمليات **باسم منفّذها الحقيقي**، والشريط الجانبي يعرض الحساب المسجّل.
- **اختبار غير مسبوق:** شغّلنا Postgres حقيقيًا داخل بيئة التطوير، ثم متصفح آلي أنشأ حسابًا ومتطلبًا من الواجهة نفسها، وتحققنا في القاعدة من الملكية والعزل والتشفير — قبل أي اعتماد.

### المرحلة ٢٠ — توجيه النماذج وتتبّع الاستخدام وتقييم SMART (v1.4.0)
- **توجيه النماذج حسب الباقة**: مجاني → Haiku، احترافي → Sonnet، الأعمال → Opus — قابل للضبط عبر `AI_MODEL_*` بلا تعديل كود. الاختيار **في الخادم فقط** حسب خطة المستخدم الحقيقية.
- **تتبّع الاستهلاك**: جدول `AiUsage` يسجّل النموذج والرموز والتكلفة التقديرية والحالة (`SUCCESS`/`FAILED`/`BLOCKED_LIMIT`/`BLOCKED_SIZE`)؛ تسجيل best-effort والفاشل لا يُحتسب من الرصيد.
- **فحوصات مسبقة** قبل كل استدعاء لـ Claude: مصادقة، حدّ، حجم، وملكية المشروع/المتطلب.
- **إعادة تعيين شهرية** لرصيد التحليلات عند أول استدعاء بعد `resetDate`.
- **تحسينات التحليل**: SMART ثلاثي الحالة (مستوفٍ/جزئي/غير مستوفٍ) مع سبب، أنواع متطلب واضحة، وحقل ملاحظات اختياري يُمرَّر للذكاء الاصطناعي.
- تحقّق: أنواع + بناء + وحدة (توجيه ٨/٨، إعادة تعيين ٦/٦) + تسجيل استخدام على Postgres حقيقي.

### المرحلة ٤٢ — أساس محاسبة الذكاء الاصطناعي والامتيازات (v2.6)
- **الهدف الأول: حماية تكلفة الـ API.** استبدلنا «عدّاد التحليلات» الواحد بنظام محاسبة كامل بأسلوب ChatGPT/Cursor: **امتيازات مركزية** (`resolveEntitlements` — لا `if plan==="PRO"` مبعثر) + **محفظة نقاط** لكل مستخدم + **سجل إلحاقي** (`AiLedgerEntry`) لا يُعدَّل + **عملية لكل طلب** (`AiOperation`) كمرساة Idempotency وسجل مراجعة كامل.
- **دورة حجز/تثبيت/استرجاع ذرّية**: الحجز يتحقق ذرّيًا أن النقاط دون المنحة (والسقف اليومي) في نفس العبارة، ومفتاح Idempotency فريد + تراجع المعاملة يضمنان **عدم الخصم المزدوج** عند النقر المزدوج/إعادة المحاولة. النجاح يثبّت، الفشل/المهلة يسترجع. اختُبر: 20 حجزًا متوازيًا × 3 نقاط على منحة 30 = **بالضبط 10 نجحت**، ونفس المفتاح مرتين = خصم واحد، والفشل يعيد الرصيد (**19/19**).
- **مستقل عن المزوّد**: طبقة المحاسبة تتعامل مع نقاط/رموز/تكلفة وسلاسل provider/model فقط؛ تبديل OpenAI/Gemini/OpenRouter = تعديل إعدادات لا كود. **كل الأرقام** (تكلفة المهام/المنح/الحدود/المهلة/الأسعار) قابلة للتعديل من الأدمن مع سقوف صلبة وسجل تدقيق.
- **لا توافق خلفي** (لا مستخدمين حقيقيين) → بنية نظيفة: أُزيل `AiUsage`، والحقول القديمة، بلا Dual-write. العميل يرى النقاط فقط (لا Tokens/أسعار/أسماء نماذج). **لم نبنِ** ملفات المشاريع/الثيمات/BRD-SRS/التعاون (مراحل لاحقة). tsc + بناء + دورة مسار حية + مسح أسرار: أخضر.

- أُغلقت ثغرتان متوسطتان (جلسة مستخدم محذوف/معطل تتجاوز القياس والملكية؛ اسم نموذج داخلي ظاهر لكل المستخدمين) وثلاث ملاحظات منخفضة — والباقي وُثّق كتوصيات مصنفة في تقرير الإطلاق.

### المرحلة ٢٦ — البقاء داخل صفحة المتطلب + تبويب «تقييم الجودة» (v1.9.4)
- إصلاح تجربة ما بعد المساعد: كان المستخدم يُعاد للقائمة الرئيسية بعد نجاح التحليل لأن تحديث البيانات يعيد بناء الواجهة — الآن تُحفظ الشاشة والمتطلب في `sessionStorage` ويُشتق المتطلب من أحدث بيانات، فتبقى المهام الست كلها داخل نفس الصفحة.
- **تبويب «تقييم الجودة»** جديد يعرض النتائج التفصيلية (SMART، الغموض والمخاطر، الصياغة المحسّنة مع الاعتماد)، واللوحة الجانبية صارت ملخصًا سريعًا وأزرار مهام فقط مع زر «عرض النتائج التفصيلية».
- رسالة نجاح واضحة «تم تحديث نتائج مساعد وثّق بنجاح.» مع فتح التبويب تلقائيًا، ورسالة خطأ بلا إخراج «تعذر تشغيل مساعد وثّق. حاول مرة أخرى.» — وحفظ التبويب النشط لكل متطلب حتى بعد إعادة التحميل.
- تحقّق: ٢١ فحص E2E بمحاكاة استجابات المساعد (نجاح/مهام خفيفة/خطأ/إعادة تحميل/حالة فارغة) + بناء نظيف.

### المرحلة ٢٥ — سلسلة إصلاحات تقييم الجودة في الإنتاج (v1.9.1 – v1.9.3)
- ثلاث طبقات أعطال متتالية ظهرت في الإنتاج وأُصلحت واحدة تلو الأخرى: **انقطاع JSON عند حد الرموز** (رفع الحدود + مستخرج موحّد بأخطاء مصنّفة في لوحة الأدمن) ← **قتل الدالة عند مهلة ٦٠ ثانية** (رفعها إلى ٣٠٠ + حدود إيجاز في التعليمات) ← **انتهاء معاملة Prisma على Neon** (دفعات `createMany` + مهلة معاملة صريحة).
- درس مستفاد: كل إصلاح كشف الطبقة التالية — التشخيص من `AiUsage.errorMessage` ولوحة الأدمن اختصر الطريق.

### المرحلة ٢٤ — وثائق BRD و SRS (v1.9.0)
- نافذة التصدير صارت تختار **نوع المستند** (تقرير تحليل / BRD / SRS) بوصف تحت كل نوع، مع PDF وWord قابل للتعديل والنطاق المفلتر.
- **BRD** بـ ١٥ قسمًا و**SRS** بـ ١٥ قسمًا (مع RTM وحالات استخدام مستنتجة موسومة «مقترحة») — كلها من بيانات المشروع الحقيقية فقط، والنواقص تُوسم صراحة «يحتاج استكمال» بدل الاختراع.
- `report-config.ts` يجمع كل النصوص القابلة للتغيير، وبنية «تحسين الصياغة بالمساعد» جاهزة لـ v2.0 (معطَّلة) — الوثائق لا تستهلك أي ذكاء اصطناعي.
- تحقّق: توليد فعلي من مشروع حقيقي بلقطات + فحص آلي (لا نماذج مذكورة، أوسمة النقص تعمل) + جوال + بناء نظيف.

### المرحلة ١٧ — صفحة الهبوط (v1.1.0)
- بناء **صفحة هبوط عربية احترافية** بمستوى SaaS عالمي (Hero، مميزات، كيف يعمل، لقطات حقيقية بتكبير، أرقام متحركة، أسئلة شائعة، CTA، تذييل) في مكوّن واحد بدون مكتبات.
- التقطنا **لقطات حقيقية** من النظام (عبر متصفح آلي) لاستخدامها في الصفحة.
- `/` صار يوجّه: الزائر → صفحة الهبوط، والمسجّل → مساحة العمل.
- تحصين أمني: حارس في كل Server Action يرفض الطلبات غير المصادَقة بعد فتح `/` للعموم.
- تحقّق بصري (سطح المكتب/جوال) واختبار توجيه كامل.

## 📚 دروس مستفادة

1. **متغيّرات البيئة تنطبق على النشرات الجديدة فقط** — بعد إضافة أي متغيّر، لازم
   **Redeploy** (ويُفضّل بدون Build Cache).
2. **الأسرار لا تدخل الكود** — المفاتيح (قاعدة البيانات، Claude) تُحفظ كمتغيّرات بيئة،
   خصوصًا والمستودع عام.
3. **Vercel Hobby = استخدام فردي** — صاحب المستودع + صاحب الـ commits + صاحب حساب
   Vercel لازم يكونون نفس الشخص، أو يكون المستودع عامًّا.
4. **التراجع الآمن يمنع الأعطال** — صمّمنا التطبيق ليعمل حتى لو غابت القاعدة أو المفتاح.
5. **التصميم الجيد للمجلدات** — التطبيق في `wathiq-app/`، فيلزم ضبط Root Directory.
