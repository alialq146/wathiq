# 🛠️ رحلة بناء وثّق — سجل التطوير

توثيق تسلسلي لرحلة بناء منصة **وثّق** من تصميم ثابت إلى منتج حيّ كامل
(واجهة + قاعدة بيانات + ذكاء اصطناعي حقيقي).

**التاريخ:** ٣٠ يونيو ٢٠٢٦ · **المطوّر:** alialq146 · **الرابط:** https://wathiq-ai.vercel.app

---

## نظرة عامة

بدأنا من **حزمة تصميم** (نماذج HTML/CSS مُصدّرة من أداة تصميم) وانتهينا بتطبيق
**Next.js + TypeScript** منشور على Vercel، مربوط بقاعدة بيانات **Neon Postgres**،
ويستخدم **Claude (Opus 4.8)** لتحليل وثائق المتطلبات فعليًا.

---

## المراحل بالتسلسل

### المرحلة ١ — من التصميم إلى تطبيق حقيقي
- قراءة حزمة التصميم كاملة: `index.html` ومستورداته، التوكنز، و١٣ مكوّنًا.
- تحويل النماذج إلى تطبيق **Next.js (App Router) + TypeScript**.
- نقل **توكنز التصميم** (ألوان، خطوط، مسافات، تأثيرات) إلى CSS — عربي أولاً (RTL).
- نقل **١٣ مكوّنًا** إلى مكوّنات React مُوثّقة الأنواع.
- بناء **الهيكل + الشاشات الأربع** (نظرة عامة، المتطلبات، التفاصيل، تحليل وثّق).
- **التحقق:** `npm run build` نجح بدون أخطاء.

### المرحلة ٢ — الرفع على GitHub
- إنشاء مستودع `alialq146/wathiq`.
- حزم المشروع ورفعه (بدون `node_modules`).
- كتابة `README.md` ثنائي اللغة (عربي + إنجليزي).

### المرحلة ٣ — تأمين Next.js
- **المشكلة:** Vercel حظر النشر لأن إصدار Next.js فيه ثغرة أمنية.
- **الحل:** الترقية إلى `15.5.19` (أحدث إصدار مُصحَّح على نفس الجيل) → **٠ ثغرات**.

### المرحلة ٤ — النشر على Vercel
- استيراد المستودع في Vercel.
- 🔑 **نقطة مهمة:** التطبيق داخل مجلد فرعي `wathiq-app/` → ضبط **Root Directory = `wathiq-app`**.
- النشر.

### المرحلة ٥ — حل مشكلة الحظر (Deployment Blocked)
- **المشكلة:** Vercel (خطة Hobby) حظر النشر لأن **صاحب الـ commits** كان باسم/إيميل
  مختلف عن صاحب حساب Vercel.
- **الحل:** إعادة كتابة هوية كل الـ commits لتصبح باسمك `alialq146`، ثم دفع إجباري.
- **النتيجة:** النشر نجح ✅.

### المرحلة ٦ — قاعدة البيانات (Neon Postgres)
- إنشاء قاعدة **Neon** من تبويب Storage في Vercel وربطها بالمشروع.
- بناء طبقة بيانات بـ **Prisma** مع **تراجع آمن**: لو ما فيه قاعدة، التطبيق يشتغل
  ببيانات افتراضية بدل ما ينكسر.
- إضافة نقطة فحص `/api/health` تكشف مصدر البيانات (`database` أو `fallback`).
- سكربت بذرة **«مرة واحدة»** حتى لا تُمسح التعديلات عند إعادة النشر.

### المرحلة ٧ — نطاق مخصص
- إضافة النطاق النظيف **`wathiq-ai.vercel.app`** من إعدادات Domains.

### المرحلة ٨ — لوحة تعديل المتطلبات
- إضافة/تعديل/حذف المتطلبات من الواجهة عبر **Server Actions** تكتب في القاعدة.
- نافذة (Dialog) للنموذج + أزرار تعديل/حذف على كل بطاقة.

### المرحلة ٩ — تحليل وثّق بالذكاء الاصطناعي
- ربط التحليل بـ **Claude (claude-opus-4-8)** عبر `@anthropic-ai/sdk`.
- **مخرجات منظّمة (Structured Outputs)** لاستخراج المتطلبات بدقة.
- لصق وثيقة → استخراج المتطلبات + ملخّص شفّاف + خطوات الاستنتاج + توصيات + درجة ثقة.
- زر **«حفظ في القاعدة»** يضيف المستخرجة إلى Neon.
- المفتاح يُقرأ من متغيّر البيئة `ANTHROPIC_API_KEY` (لا يدخل الكود أبدًا).

### المرحلة ١٠ — رفع PDF للتحليل
- دعم رفع ملف **PDF** وتحليله مباشرةً عبر Claude (كتل محتوى مستند base64).
- واجهة سحب/إفلات مع حدّ حجم آمن يتوافق مع حدود Vercel للطلبات.

### المرحلة ١١ — البحث والتصفية والترتيب
- بحث برقم المتطلب أو نصّه، وفلترة بالحالة والأولوية، وترتيب — مع زر «مسح كل الفلاتر».
- ربط حقل البحث في الشريط العلوي بقائمة المتطلبات.

### المرحلة ١٢ — سير الاعتماد
- تفعيل زرّي **«طلب الاعتماد»** و**«طلب معلومات إضافية»** ليغيّرا حالة المتطلب فعليًا في القاعدة.
- حالة تحميل + رسائل نجاح/خطأ + تحديث فوري لشارة الحالة في التفاصيل والقائمة.

### المرحلة ١٣ — دورة المتطلب الكاملة والسياق وسجل التدقيق
- ربط معايير القبول/القواعد/الأسئلة بمتطلب محدّد وعرضها مفلترة.
- **إضافة معيار قبول**، **تبديل الإنجاز**، و**الإجابة عن الأسئلة** من الواجهة (Server Actions).
- **رؤى الذكاء الاصطناعي** صارت مبنية على بيانات المتطلب الحقيقية.
- بناء صفحات **أصحاب المصلحة** و**قواعد العمل** و**سجل التدقيق** بشكل حقيقي.
- **سجل تدقيق (AuditEvent)** يوثّق كل تغيير كخط زمني.
- تفعيل **مبدّل المشروع** و**قائمة الإعدادات** المنبثقة.

### المرحلة ١٤ — تصدير المتطلبات
- زر **«تصدير»** بثلاث صيغ: **PDF** (عبر طباعة المتصفح)، **Word** (‏.doc)، **Excel** (‏.csv).
- بدون مكتبات خارجية لضمان دعم العربية/RTL المثالي وخفّة الحزمة.
- تم التحقق البصري من مخرج PDF عبر لقطة شاشة (Chromium) قبل الدمج.

### المرحلة ١٥ — تسجيل الدخول
- صفحة دخول (بريد + كلمة مرور) بجلسة عبر **Cookie موقّع (HMAC)** + حماية بالـ **middleware**.
- **آمن بالتصميم:** المصادقة معطّلة ما لم تُضبط متغيّرات البيئة، فلا تُقفل الموقع بالخطأ.
- تحقّق شامل: تحويل غير المُصادَق إلى `/login`، رفض/قبول بيانات الدخول، ١١ اختبار وحدة، ولقطة شاشة للشاشة.

### المرحلة ١٦ — نظام الحسابات الكامل (v1.0.0)
- جدول **User** + صفحة **إنشاء حساب** `/signup` — أي شخص يسجّل ويعمل في مساحته.
- كلمات المرور مشفّرة بـ **scrypt**، والجلسات بـ Cookie موقّع يحمل هوية المستخدم.
- **عزل البيانات:** `ownerId` على كل الجداول؛ كل حساب يرى بياناته + التجريبية المشتركة، وحُماية الملكية مفروضة في كل Server Action.
- سجل التدقيق يوثّق العمليات **باسم منفّذها الحقيقي**، والشريط الجانبي يعرض الحساب المسجّل.
- **اختبار غير مسبوق:** شغّلنا Postgres حقيقيًا داخل بيئة التطوير، ثم متصفح آلي أنشأ حسابًا ومتطلبًا من الواجهة نفسها، وتحققنا في القاعدة من الملكية والعزل والتشفير — قبل أي اعتماد.

### المرحلة ١٨ — جاهزية النشر (v1.2.0)
- **خطة مجانية** (تحليل واحد) + بطاقة ترقية عبر **واتساب** + صفحة أسعار `/pricing`.
- **وضع داكن** كامل عبر توكنات المظهر + زر تبديل بلا وميض.
- **صفحات قانونية** (خصوصية/شروط) + **Favicon** و**OpenGraph**.
- **المستخدم الجديد يبدأ فارغًا** + شاشة نظرة عامة مبنية على بيانات حقيقية.
- **توافق كامل مع الجوال** (قائمة منزلقة + انسدال لوحة التفاصيل + شبكات متكيّفة).
- تحقّق بصري شامل (سطح مكتب/جوال، فاتح/داكن) عبر لقطات Chromium.

### المرحلة ١٧ — صفحة الهبوط (v1.1.0)
- بناء **صفحة هبوط عربية احترافية** بمستوى SaaS عالمي (Hero، مميزات، كيف يعمل، لقطات حقيقية بتكبير، أرقام متحركة، أسئلة شائعة، CTA، تذييل) في مكوّن واحد بدون مكتبات.
- التقطنا **لقطات حقيقية** من النظام (عبر متصفح آلي) لاستخدامها في الصفحة.
- `/` صار يوجّه: الزائر → صفحة الهبوط، والمسجّل → مساحة العمل.
- تحصين أمني: حارس في كل Server Action يرفض الطلبات غير المصادَقة بعد فتح `/` للعموم.
- تحقّق بصري (سطح المكتب/جوال) واختبار توجيه كامل.

## 📚 دروس مستفادة

1. **متغيّرات البيئة تنطبق على النشرات الجديدة فقط** — بعد إضافة أي متغيّر، لازم
   **Redeploy** (ويُفضّل بدون Build Cache).
2. **الأسرار لا تدخل الكود** — المفاتيح (قاعدة البيانات، Claude) تُحفظ كمتغيّرات بيئة،
   خصوصًا والمستودع عام.
3. **Vercel Hobby = استخدام فردي** — صاحب المستودع + صاحب الـ commits + صاحب حساب
   Vercel لازم يكونون نفس الشخص، أو يكون المستودع عامًّا.
4. **التراجع الآمن يمنع الأعطال** — صمّمنا التطبيق ليعمل حتى لو غابت القاعدة أو المفتاح.
5. **التصميم الجيد للمجلدات** — التطبيق في `wathiq-app/`، فيلزم ضبط Root Directory.
