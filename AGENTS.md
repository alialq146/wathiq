## 1. ما هذا المشروع

**وثّق (Wathiq)** — منصة SaaS عربية-أولاً (RTL) لتحليل متطلبات الأعمال. التطبيق في `wathiq-app/`.
المكدّس: Next.js 15 (App Router) + React 19 + TypeScript + Prisma/Postgres (Neon) + مزوّد ذكاء اصطناعي اختياري، على Vercel.

## 3. سير العمل القياسي (لكل إصدار)

1. **افحص الملفات الفعلية** — لا تفترض. اقرأ قبل أن تعدّل.
2. **نفّذ على مراحل**؛ بعد كل مرحلة: `npx tsc --noEmit` + `npx next build`.
3. **وثّق كل إصدار**: حدّث `CHANGELOG.md` + `JOURNEY.md` (جذر المستودع) + ارفع `wathiq-app/src/lib/version.ts`.
4. **حدّث الوثائق المتأثّرة** في `wathiq-app/docs/` (خصوصًا `CURRENT_STATUS.md`).
5. **الفحص قبل الدفع**: tsc نظيف + build ناجح + مسح أسرار + انحدار سريع.
6. **الالتزامات موقّعة**؛ الدفع لا يتم إلا بطلب صريح من المستخدم.

### تعريف الإنجاز (Definition of Done) — قبل إغلاق أي مهمة

- [ ] **تنفيذ الميزة مكتمل** ويعمل كما هو مقصود.
- [ ] **فحص TypeScript ناجح** (`npx tsc --noEmit`).
- [ ] **بناء Next.js ناجح** (`npx next build`).
- [ ] **الوثائق المتأثّرة فقط حُدِّثت** (لا تلمس ما لا علاقة له بالتغيير — القاعدة 13).
- [ ] **روابط الوثائق مُتحقَّقة** بعد أي تعديل توثيق (القاعدة 14).
- [ ] **`CHANGELOG` + `JOURNEY` + `version.ts` حُدِّثت عند الحاجة** (للإصدارات — الخطوة 3). **تغييرات التوثيق وحدها لا تتطلّب رفع إصدار.**
- [ ] **لا أسرار مكشوفة** (القاعدة 5).
- [ ] **حوكمة المستودع محترمة** (القواعد غير القابلة للتفاوض §4).

**«الانحدار السريع» = قائمة دخان يدوية** (لا يوجد suite آلي — انظر `wathiq-app/docs/TESTING.md`):
تسجيل دخول → إنشاء مشروع → إضافة/تعديل متطلب → تشغيل تحليل (إن كان المفتاح مضبوطًا) → تصدير → التأكد من عزل حساب آخر لا يرى بياناتك. أي انكسار = أوقف الدفع.

## 4. القواعد غير القابلة للتفاوض

1. **افحص الملفات الفعلية، لا تفترض.**
2. **لا تغيّر سلوكًا وظيفيًا** إلا لإصلاح خطأ/ثغرة مُثبتة.
3. **لا refactor واسع** دون طلب — اكتبه كتوصية.
4. **لا تقنية جديدة** (Redis/Queue/تخزين…) دون حاجة مثبتة — انظر ADRs.
5. **لا تكشف أو تطبع أسرارًا** — أسماء المتغيّرات فقط، لا قيم.
6. **لا تذكر أسماء نماذج الذكاء أو المزوّد** في الواجهة أو التقارير أو الكود.
7. **لا تحذف بيانات إنتاج ولا تعيد تهيئة القاعدة.**
8. **الهجرات آمنة/متوافقة/قابلة للمراجعة** — انتبه لـ `db push --accept-data-loss` (استخدم `SKIP_DB_PUSH` في الإنتاج).
9. **حافظ على العزل**: مرّ عبر `requireActor`/`ownedBy`/`scopeWhere`/`requireSuperAdmin` — لا تثق بمدخلات العميل للملكية/الدور.
10. **حافظ على RTL + الجوال + الوضعين (فاتح/داكن).**
11. **القيم السياسية من الإعدادات، الأسرار من متغيّرات البيئة.**
12. **لا تدّعِ أمانًا 100%** — استخدم تصنيف المخاطر.
13. **مصدر واحد للحقيقة + لا تكرار توثيق** — كل موضوع يُوثَّق في ملفٍ مرجعيٍّ واحد؛ الملفات الأخرى **تربط** إليه ولا تنسخ محتواه. لا تُنشئ وثيقةً جديدة لموضوعٍ موثَّقٍ مسبقًا.
14. **بعد أي تعديل توثيق: تحقّق من الروابط** — لا روابط مكسورة داخل ملفات `wathiq-app/docs/` أو بين ملفات الجذر (`README`/`AGENTS`/`CHANGELOG`…) والوثائق.
15. **أبلِغ عن التوثيق القديم/المتعارض، ولا تتجاهله بصمت** — إن وجدت وثيقةً تصف سلوكًا لم يعد قائمًا، سجّلها كبند تصحيح لاحق؛ لا تُصلحها خارج نطاق مهمتك ولا تتركها دون ذكر.
16. **كل قرار معماري يغيّر اتجاه المشروع يُوثَّق كـ ADR** في `wathiq-app/docs/DECISIONS/` (بنفس نمط الـ ADRs القائمة: السياق/القرار/البدائل/النتائج) مع تحديث فهرس `DECISIONS/README.md`.
17. **كل منطق عمل قابل للتهيئة يجب أن يبقى قابلًا للتهيئة** (توسيعٌ صريح للقاعدة 11) — القيم المتوقَّع تغيّرها مع الزمن **لا تُقسّى في الكود أبدًا**. أمثلة (لا حصر): نماذج الذكاء الاصطناعي، الأسعار، باقات الاشتراك، حدود الاستخدام، أعلام الميزات (feature flags)، رقم واتساب، قوالب البريد، حدود حجم الملفات، حدود المعدّل (rate limits)، سياسات إعادة المحاولة، ومهل الانتظار (timeouts). **مواقع التهيئة المفضّلة**: متغيّرات البيئة، أو إعدادات الأدمن (`src/lib/settings/`)، أو إعدادات الخادم — بحيث لا تتطلّب تغييرات العمل الاعتيادية تعديل كود.

## 5. قواعد الكود

- **الكتابات** عبر Server Actions في `wathiq-app/src/app/actions.ts`، كلها مُنطّقة بالملكية.
- **مسارات الأدمن** (`/api/admin/**`): `requireSuperAdmin()` أولًا.
- **الإخراج النصي** (CSV/HTML): استخدم `csvCell`/`esc` في `src/lib/export.ts`.
- **المدخلات النصية**: طبّق حدود طول (نمط `cap()` في actions.ts) + قوائم بيضاء للحقول المُعدّدة.
- **الأنماط**: طابِق الكود المحيط (كثافة التعليقات، التسمية، RTL، توكنز التصميم في `src/styles/tokens/`).
- **الإعدادات** تُقرأ عبر `src/lib/settings/` (مع cache) — لا تُقسّي قيمًا سياسية في الكود.

## 6. التشغيل السريع

```bash
cd wathiq-app
npm install
npm run dev                      # بلا DATABASE_URL → بيانات احتياطية
npx tsc --noEmit && npx next build   # فحص إلزامي قبل الدفع
```

قاعدة محلية، متغيّرات، نشر: `wathiq-app/docs/DEPLOYMENT.md` + `ENVIRONMENT_VARIABLES.md` + `TESTING.md`.
