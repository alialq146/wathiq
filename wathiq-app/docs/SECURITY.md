# الأمان · Security

> تصنيف المخاطر الدفاعي ونتائج التدقيق الأمني (Phase C من تدقيق v2.5).
> **لا ادّعاء بأمان 100%** — هذا تصنيف مخاطر شفّاف. آخر تدقيق: v2.5.0 (2026-07).

## 1. الوضع الحالي (ملخّص)

لا توجد ثغرات **حرجة (Critical)** أو **عالية (High)** مفتوحة بعد إصلاحات v2.5.
البنود المتوسطة/المنخفضة إمّا عولجت أو وُثّقت كتوصيات مقبولة المخاطر.

## 4. ترويسات الأمان (v2.5)

مُطبَّقة على كل المسارات (`next.config.mjs`):
`X-Content-Type-Options: nosniff` · `X-Frame-Options: SAMEORIGIN` · `Referrer-Policy: strict-origin-when-cross-origin` · `Permissions-Policy: camera=(), microphone=(), geolocation=()` · `Strict-Transport-Security: max-age=31536000; includeSubDomains`.

> **ملاحظة**: لا CSP صارمة عمدًا (تجنّب كسر Next inline scripts). إضافتها لاحقًا تتطلّب nonce — انظر ROADMAP.

## 6. قواعد ذهبية للمساهمين

1. **لا تكشف أسرارًا** ولا تطبع قيم مفاتيح — أسماء المتغيّرات فقط.
2. **لا تذكر أسماء نماذج الذكاء أو المزوّد** في الواجهة أو التقارير.
3. لا رسائل خطأ تقنية للمستخدم — رسائل عامة (حدود الأخطاء تتكفّل).
4. أي مسار حسّاس جديد: مرّ عبر `requireActor`/`ownedBy`/`requireSuperAdmin`.
5. أي إخراج نصي (CSV/HTML): استخدم `csvCell`/`esc`.
6. أي مدخل نصي: طبّق حدود طول (نمط `cap()` في actions.ts).
7. لا تُعطّل فحص الملكية من الإعدادات أبدًا.

## 7. الإبلاغ عن ثغرة

راجع صفحة `/security` العامة لسياسة الشفافية. داخليًا: صنّف الخطورة (حرجة/عالية/متوسطة/منخفضة/وقائية)، أرفق دليلًا قابلًا لإعادة الإنتاج، وأصلح الحرجة/العالية قبل الإطلاق.
