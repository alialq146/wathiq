# ADR-013: محاسبة الذكاء الاصطناعي بالنقاط (Credit Ledger)

**الحالة**: مُعتمد · **السياق**: v2.6

## السياق
«عدّاد التحليلات» الواحد (v2.5) لم يعد يعكس تعدد مهام المساعد، ولا يحمي التكلفة بدقة، وبلا Idempotency (خصم مزدوج ممكن) ولا سجل مراجعة.

## القرار
نظام محاسبة كامل: امتيازات مركزية (`resolveEntitlements`) + محفظة نقاط لكل مستخدم + سجل إلحاقي (`AiLedgerEntry`) + عملية لكل طلب (`AiOperation`، مرساة Idempotency) + دورة حجز/تثبيت/استرجاع ذرّية. لا `if plan==="PRO"` مبعثر. طبقة المحاسبة مستقلة عن أي مزوّد ذكاء اصطناعي.

## البدائل المرفوضة
- إبقاء العدّاد + إضافة أوزان: لا يحل Idempotency ولا السجل ولا الاستقلال عن المزوّد.
- سجل خالص بلا محفظة مادّية: مجموع O(n) لكل فحص — غير قابل للتوسّع.

## النتائج
- (+) حماية تكلفة حقيقية (حجز ذرّي ضد المنحة + سقف يومي + سقف عملية)، لا خصم مزدوج (مفتاح فريد + تراجع المعاملة)، سجل تدقيق كامل، كل الأرقام قابلة للتعديل من الأدمن، تبديل المزوّد بلا تعديل محاسبة.
- (−) نموذجان جديدان + محفظة مادّية = تجسيد للسجل (يجب أن يبقيا متسقين — تُحدَّثان في نفس المعاملة).
- منح النقاط عند التفعيل/التجديد يتم في `billing.ts`؛ حتى الباقة المؤسسية لها منحة عالية-محدودة (حماية التكلفة).
