# ADR-003: بلا Redis

**الحالة**: مُعتمد

## القرار
لا Redis. التخزين المؤقت للإعدادات في ذاكرة العملية (60s) + `React cache()` لكل طلب. الحصص عبر تحديث ذرّي في Postgres.

## البدائل المرفوضة
- Redis للـ cache/rate-limit/الحصص: بنية تحتية ودَيْن تشغيلي إضافيان بلا حاجة مثبتة عند حجم الإطلاق.

## النتائج
- (+) بنية أبسط، تكلفة أقل.
- (−) الـ cache لكل نسخة (instance) لا مشترك — مقبول لإعدادات نادرة التغيّر.
- مراجعة عند الحاجة لـ rate-limit موزّع أو cache مشترك واسع.
