# ADR-001: بلا علاقات Foreign-Key

**الحالة**: مُعتمد · **السياق**: v1.x فصاعدًا

## السياق
Prisma يدعم `@relation` مع cascade. لكن العزل متعدد المستأجرين يتطلّب تنطيقًا صارمًا بالملكية.

## القرار
لا `@relation` ولا `onDelete` في المخطّط. كل نموذج مُنطَّق يحمل `ownerId`/`projectId` **نصية** + `@@index`. العزل يتم في طبقة التطبيق عبر `scopeWhere`/`ownedBy`.

## البدائل المرفوضة
- علاقات FK كاملة مع cascade: تربط منطق العزل بالقاعدة، وتُصعّب التطوّر التدريجي للمخطّط، وتتعارض مع `db push`.

## النتائج
- (+) إضافة الميزات = جداول/أعمدة إضافية بلا refactor (انظر ROADMAP).
- (+) توافق تام مع `db push` والتطوّر التدريجي.
- (−) لا cascade → صفوف يتيمة عند حذف مستخدم (نادر، يُدار بسكربت تنظيف).
- (−) أي استعلام ينسى شرط الملكية = تسريب بيانات — لذا القاعدة الذهبية في ACCESS_CONTROL.
