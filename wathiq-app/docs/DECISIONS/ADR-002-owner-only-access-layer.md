# ADR-002: مالك واحد + طبقة وصول مركزية

**الحالة**: مُعتمد · **السياق**: v2.4

## القرار
النموذج single-owner (مالك واحد لكل مشروع). لكن كل فحوص الوصول تمرّ عبر طبقة مركزية `src/lib/access.ts` تُصدر عقدًا موحّدًا `ProjectAccess`، مع مصفوفة `ROLE_CAPS` جاهزة لأدوار مستقبلية.

## البدائل المرفوضة
- فحص ملكية مبعثر في كل مسار: يُصعّب إضافة التعاون لاحقًا (تعديل عشرات المواضع).
- بناء نظام فرق كامل الآن: تعقيد غير مبرَّر قبل الحاجة.

## النتائج
- (+) التعاون متعدد المستخدمين يُضاف في ملف واحد لاحقًا (جدول `ProjectMember` + توسيع `scopeWhere`).
- (+) نقطة تدقيق أمني واحدة.
- (−) طبقة تجريد إضافية قبل استخدام كل قدراتها.
