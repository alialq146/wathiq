# ADR-007: prisma db push بدل ملفات migration

**الحالة**: مُعتمد (مع تحفّظ تشغيلي)

## القرار
تطوّر المخطّط عبر `prisma db push` (بلا مجلد migrations). يُشغَّل وقت البناء عبر `scripts/db-setup.mjs`.

## البدائل المرفوضة
- `prisma migrate` بملفات: أثقل لسرعة التطوّر في هذه المرحلة.

## النتائج
- (+) تطوّر سريع للمخطّط.
- (−) **`--accept-data-loss` يعمل كل بناء إنتاجي** — خطر على بيانات الإنتاج عند تغيير مُدمِّر.
- تخفيف (v2.5): مخرج `SKIP_DB_PUSH=1` لإدارة المخطّط يدويًا في الإنتاج + نسخة احتياطية قبل التغيير.
- **مراجعة موصى بها**: اعتماد migrations مُدارة للإنتاج عند نضوج المنتج.
